# S1 — 단일 제품 평가 리포트 (ASCII 와이어프레임 v2)

## v2 컨셉

> *"전문가 친구가 옆에서 차근차근 설명해주는 평가 리포트"*

- v1 의 "논문 같은 SaaS" 톤이 너무 차가워서 일반 사용자에게 멀게 느껴짐 → v2는 **데이터·근거는 그대로 유지하되 말투·라벨·여백을 친근하게**.
- 신뢰감의 근원: 출처 카드 + 정확한 인용 (변경 없음).
- 친근감의 근원: 자연어 라벨 + 사람 말투 + 결과 우선 (Abstract → Method → Result 가 아닌, *Result + Why* 패턴).
- 데스크톱 16:9. 좌 2/3 = 분석 + 추천 / 우 1/3 = 출처·인용 사이드

## 무엇은 그대로 유지했나 (DILAB 차별화 핵심은 손대지 않음)

- ★1 **출처 카드 우측 사이드 고정** — 라벨만 친숙하게, 위치·역할·항상 노출 정책 유지
- 모든 추천에 **"AI 합성, 근거는 출처 확인"** 디스클레이머
- 5축 레이더의 *카테고리 평균 대비 위치* — 신뢰감 근거
- 감성의 *근거 사유* 노출 — 단순 비율이 아닌 *왜* 알려줌
- 여정 4단계의 *언급 수 + 감성 비율* — 일화가 아닌 데이터

→ 친숙해진 라벨 아래 *신뢰의 데이터*가 그대로 있음. 디자인 안전선(싱클리 모방 X) 유지.

## 사용자 확인 포인트 (v2)

1. **친숙도 수준이 적절한가** — 이모지·말투·"당신에게?" 같은 직접 어필이 적절한지, 너무 가벼워졌는지?
2. **5축 레이더 축 명칭** — 효능 / 성분 / 가격 / 사용감 / 안전성 그대로 OK?
3. **여정 단계 라벨** — "처음 알게 됐을 때 / 살까말까 고민 / 직접 써본 후 / 또 살까?" 가 그 동안 잘 들리나, 아니면 더 짧게 가야 하나?
4. **육각형 vs 오각형** — v1 미해결 사항 그대로. 5축이면 정오각형. 6축으로 늘리면 (+ 지속력 또는 + 친환경) 정육각형. 어느 쪽?
5. **액센트 색 한 가지** — 코랄(따뜻함) vs 머스타드 옐로우(노트북 가독성) — Level 2 HTML 목업에서 둘 다 보여드릴 수 있음.

위 5가지에 의견 주시면 Level 2 HTML 목업(실제 색·폰트·간격 적용된 이미지)으로 진행하거나 추가 조정.
