## 1. 페이지별 정보 흐름 요약

### 1.1 메인 (`/`)

11블록으로 구성된 *세로 스토리텔링* 페이지.

```
1. 헤드라인 + 가치 제안 한 줄                     ← 가치 진입
2. 1차/2차 CTA 짝 ("도입 문의" + "소개서 보기")
3. 신뢰 신호: YC 배지 + "글로벌 리더들" 로고 영역  ← 권위 부여
4. 사용 사례 분기 카드 2개                         ← 진입점 분기
   ("리서치 필요" vs "인플루언서 찾기")
5. 페인 카드 3개 (각 카드에 구체 불편함 3개씩)     ← 문제 제시
6. 솔루션 카드 3개 (각 카드에 체크박스 보조 2개)   ← 페인과 1:1 매칭
7. 실제 분석 리포트 쇼케이스 3개                   ← 진짜 결과물 노출
8. 컨설팅 옵션 (Research Partners) — 가치 3개      ← 서비스 분기
9. 고객 인용 3개 (회사명 + 부서 + 구체 문장)       ← 사회적 증거
10. 마무리 CTA ("더 이상 직감 의존 말고...")
11. 풋터 (서비스/산업별/사용 사례/리소스 4분류)
```

핵심 관찰:
- **페인 → 솔루션 1:1 매트릭스** 가 페이지의 중심. 사용자가 *자기 문제* 를 페인 카드에서 찾고 *대응 솔루션* 을 바로 옆에서 확인하는 구조.
- **YC 배지 + 큰 고객사 로고 + 실제 리포트** 3계층 신뢰 신호. 특히 *실제 분석 리포트* 노출은 강력 — "이게 정말 동작한다"를 추상 데모 없이 증명.
- 모든 CTA 가 "도입 문의" 또는 "소개서 보기" — **자가 가입 / 가격표 직접 결제 / 무료 체험** 흐름이 **메인에 없음**.

### 1.2 제품 페이지 (`/product/syncly`)

```
1. 헤드라인: "No.1 소셜 리스닝 솔루션, 싱클리"
2. CTA 짝 (메인과 동일)
3. 활용 시나리오 3개                              ← "언제 쓰나"가 먼저
   ("브랜드 위치" / "캠페인 성과" / "문제 대응")
4. 4단계 워크플로:
   ① 수집 (소셜 내용을 한곳에)
   ② 자동 정리 (3개 서브: 자동 태깅 / 분류 / 긍부정)
   ③ 인사이트 발견 (4개 서브: Hey Syncly / 트랜딩 / 익스플로어 / 검색)
   ④ 액션 연결 (2개 서브: 대시보드&리포트 / 워크플로우)
5. 자동화된 리포트 3가지 (스마트 브리프 / 교차 분석 / 리포트)
6. 리포트 쇼케이스 (메인 재활용)
7. 고객 인용 (메인 재활용)
8. FAQ 3개 (요약·외국어·고객센터 데이터)
9. 마무리 CTA + 풋터
```

핵심 관찰:
- **"무엇을 할 수 있나(What)" 보다 "언제 쓰나(When)" 가 먼저** — 활용 시나리오 3개가 페이지 상단.
- **4단계 워크플로 = 페이지 정보 구조 자체** — 사용자가 *내부 동작* 을 따라가며 이해. 추상 기능 나열이 아니라 *흐름 가시화*.
- FAQ 가 페이지 안에 *통합* — 별도 헬프 센터로 보내지 않음.

### 1.3 가격 페이지 (`/pricing`)

```
1. 헤드라인 + 1줄 카피 ("합리적인 가격, 그 이상의 가치")
2. 2단 카드:
   - Pro: 75만원~/월 (연 결제, VAT 별도, 에디터 3명까지)
     기능 4줄 + "도입 문의하기" CTA
   - Enterprise: 가격 문의 (Pro + 2개 이상 툴·API·전담 매니저)
     "도입 문의하기" CTA
3. FAQ 4개 (Enterprise 요금 / 데모 / 할인 / 결제)
4. 마무리 CTA + 풋터
```

핵심 관찰:
- **가격은 공개하지만 결제 흐름 X** — 모든 CTA 가 "도입 문의". 자가 결제·카드 등록·무료 체험 *전혀 없음*.
- 단 2티어 (Pro / Enterprise). 학생·소규모 브랜드용 진입 티어 부재.

## 2. 정보 제공 방식 패턴 (6개)

### A. **사용 시나리오 우선 (When > What)**

"무엇을 할 수 있나" 보다 "언제 쓰나" 가 페이지 상단. 사용자가 *자기 상황* 을 찾고 그 다음 *기능* 을 본다.

### B. **페인-솔루션 1:1 매트릭스**

페인 카드 3개 → 솔루션 카드 3개 1:1 대응. 페인 카드에 *구체 bullet 2-3개* + 솔루션 카드에 *체크박스 보조 2개* — 매핑이 정보 구조로 가시화.

### C. **신뢰 신호 4계층**

1. **외부 권위** ("Backed by Y Combinator" 배지)
2. **고객 로고 영역** (큰 브랜드 노출)
3. **실제 분석 리포트 쇼케이스** (← *가장 강력*. 추상 데모 X, 진짜 결과물 노출)
4. **고객 인용** (회사명 + 부서 + 구체 문장 + 수치)

### D. **워크플로 = 페이지 정보 구조**

수집 → 정리 → 인사이트 발견 → 액션 4단계가 페이지 자체의 흐름. 추상 기능 카드가 아니라 *사용자가 따라가는 여정*.

### E. **모든 CTA = 영업 컨택**

가격 페이지에서도 "도입 문의하기" 만. 자가 가입·무료 체험·카드 결제 흐름 부재 → **B2B sales-led 진입 정책**.

### F. **풋터에 수직 도메인 페이지 분기**

산업별: 뷰티 / F&B 페이지 따로. 사용 사례별 페이지 따로. *도메인-플렉시블 마케팅* 의 표준 패턴.

## 3. DILAB 에 차용할 만한 정보 흐름 (시각 X, 패턴만)

| # | 차용 패턴 | DILAB v2 적용 방식 |
|---|---|---|
| **차용-1** | 활용 시나리오 우선 | DILAB 메인에 "협력사 대표가 정부 R&D 제안서 쓸 때" / "리서치 팀이 시장 동향 볼 때" 시나리오 카드 2-3개 (When 먼저) |
| **차용-2** | 페인-솔루션 1:1 매트릭스 | DILAB 의 3대 페인 ("리뷰 신뢰성 X" / "출처 추적 X" / "전문가 관점 부재") ↔ 솔루션 ("전문가 DB + RAG" / "출처 카드 항상" / "도메인 큐레이팅") 1:1 |
| **차용-3** | 실제 분석 리포트 쇼케이스 | DILAB 도 화장품 도메인으로 *실제 리포트 1-2건* 만들어서 가입 전 공개 노출 — 가장 강력한 신뢰 신호 |
| **차용-4** | 워크플로 4단계 = 페이지 구조 | DILAB 제품 페이지: ① 수집 → ② 임베딩 → ③ RAG 검색 → ④ **출처 명시 답변** 4단계. 마지막 단계가 **DILAB 차별화 강조 슬롯** |
| **차용-5** | 고객 인용에 회사+부서+구체 문장 | DILAB 도 전문가 인용에 *이름·전문 분야·신뢰도* 표기 (S1 목업에 이미 적용됨 — 강화) |
| **차용-6** | 수직 도메인 페이지 | "화장품 / F&B / 가전" 도메인별 페이지를 *향후* 마련. ★2 도메인-플렉시블의 마케팅 표현 |
| **차용-7** | FAQ 페이지 내 통합 | DILAB 도 별도 헬프 X. 가격·제품 페이지 안에 FAQ 통합 (대학원생이 운영 부담 적음) |

## 4. DILAB 이 *반대로 갈* 정보 흐름 (의도적 차별화)

| # | 싱클리 | DILAB 차별화 |
|---|---|---|
| **차별-1** | 가격 페이지 모든 CTA = 영업 컨택 | **자가 가입 + freemium** ([PRD G4](../prd/dilab-mvp-prd.md)). 가격 페이지에 *바로 시작하기* 버튼 |
| **차별-2** | 외부 권위 (YC 배지) + 큰 고객사 로고 | DILAB 은 외부 권위 약함 (학생 MVP). 대체 신뢰 신호 = **전문가 인용 깊이 + 출처 카드의 *외부 노출*** |
| **차별-3** | Hey Syncly = 답변만 (출처는 제품 안) | **DILAB Ask = 답변 + 출처 카드 항상 명시** — 마케팅 화면에서도 *출처 패턴* 노출. 차별화가 마케팅 단에서부터 시작 |
| **차별-4** | 답변의 가치 = "*어떻게* 분석" (방법론) | DILAB 의 가치 = "*어떻게* + *어디 근거*" (방법론 + 출처). 마케팅 카피에서부터 *출처 추적* 을 1차 가치로 |
| **차별-5** | 2티어 (Pro 75만원~ / Enterprise) | **3티어** (Free / Pro 5-15만원 / Custom). 학생·연구실·소규모 브랜드용 진입 티어 추가 |

## 5. v2 HTML 목업에 *추가* 할 수 있는 정보 흐름 (S2~S5 설계 시)

S1 (단일 제품 리포트) 은 이미 작성. 차용 패턴을 어디 적용할지:

| 패턴 | 어느 화면에 |
|---|---|
| 차용-1 (시나리오 우선) | S0 — *랜딩 페이지* (현재 미설계). DILAB Ask 진입 전에 시나리오 분기. 또는 S2 (Ask) 상단에 *예시 질문 카드* 로 |
| 차용-2 (페인-솔루션 매트릭스) | S0 랜딩 또는 *About* 페이지 |
| 차용-3 (실제 리포트 쇼케이스) | S0 랜딩 — "이런 리포트가 나옵니다" 섹션. S1 으로 진입하기 전 *맛보기* |
| 차용-4 (워크플로 4단계) | S0 또는 *제품 소개* 페이지. 또는 S2 (DILAB Ask) 의 *답변 생성 중* 진행 표시에 "수집 → 검색 → RAG → 출처 정렬" 4단계 progress |
| 차용-5 (고객 인용 상세) | S1 우측 사이드 (이미 적용됨, 강화 검토) |
| 차별-3 (출처 카드 마케팅 단 노출) | S0 랜딩 — *DILAB Ask 데모* 영역에서 답변 카드와 출처 카드를 *함께* 보여줌 |

→ **S0 (랜딩 페이지) 가 새로 필요** 함이 가장 큰 발견. 기존 계획서 5화면 (S1~S5) 에는 *랜딩* 이 없었음.
