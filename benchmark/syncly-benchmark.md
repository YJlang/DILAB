## 🔑 한 문장 요약 (Executive)

싱클리는 "Sales-led 한국 SaaS"로 **다채널 VoC를 한 곳에 모아 자동 분류·감성 분석·자연어 질의("Hey Syncly")로 답하는 통합 인텔리전스 플랫폼**이며, MongoDB Atlas Vector Search 기반 RAG 패턴이 핵심 기술 골격이다. **DILAB은 같은 RAG 골격을 유지하되 데이터 소스를 "전문가 큐레이팅 리뷰 DB"로 차별화**하면 "양 vs 질"의 명확한 포지셔닝이 가능하다.

> **교수님 질의 즉답**: 싱클리 직접 사용에는 **즉시 구독료가 필요하지 않다** — 단 자가 가입(self-serve)이 없고 *반드시 영업 컨택(데모 신청)* 을 거쳐야 약 1주 무료 체험을 받을 수 있으며, 그 후 Pro 75만원~/월 또는 Enterprise 견적제로 전환된다 [4][5][6][30].

### 5.2 단계별 스택 (공식 + 추정)

| 단계 | 기능 | 기술 | 자체 vs 외부 | 확신도 |
|---|---|---|---|---|
| 수집 | 다채널 크롤링 | 자체 Scalable Scraper | 자체 | 높음 |
| 전처리 | 다국어 정규화 | NLP 파이프라인 | 자체 추정 | 중 |
| 감성분석 | 긍/부정/중립 | 자체 NLP 모델(한국어 특화 추정) | 자체 | 높음 |
| 토픽 분류 | 자동 태깅 | 5-카테고리 분류기 + 사용자 정의 | 자체 | 높음 |
| 의미 클러스터링 | Semantic Unit Analysis | 임베딩 + 계층 클러스터링 추정 | 자체+MongoDB | 높음 |
| 벡터 검색 | 유사도 매칭 | **MongoDB Atlas Vector Search** | 외부 | 높음 |
| 자연어 질의(Hey Syncly) | 대화형 답변 | 임베딩 + **RAG** + LLM | LLM은 외부 API 추정 [unverified] | 중 |
| 요약/액션 | 인사이트·액션아이템 | LLM 생성 | 외부 API 추정 [unverified] | 낮 |

### 5.3 추정의 근거와 한계

- Hey Syncly LLM 벤더는 공식 미공개. OpenAI·Anthropic 등 외부 API 사용은 한국 SaaS 일반 패턴이며 *추정* 수준 [unverified]
- 자체 모델 fine-tuning 여부도 비공개
- 한국어 정확도/recall/precision 등 정량 지표 비공개

### 5.4 DILAB의 RAG 차별화 위치 (★ 중요)

| 축 | Syncly | DILAB(설계) |
|---|---|---|
| **RAG 데이터 소스** | 자동 수집한 **고객 피드백/리뷰** | **전문가 큐레이팅 리뷰 DB** |
| 데이터 강점 | 양·실시간성·다국가 커버리지 | **신뢰성·전문성·일관성** |
| 검색·임베딩 | MongoDB Atlas Vector Search | 같은 패턴 차용 가능(MongoDB/Pinecone/Weaviate/Qdrant 후보) |
| LLM | 외부 API 추정 | 외부 API 시작 → 도메인 fine-tuning 단계적 검토 |
| HIL(휴먼 인 더 루프) | "Research Partners" 컨설팅 번들 (서비스로 분리) | **제품 안에 전문가 검증 흐름 내장**으로 차별화 |

→ 같은 RAG 골격이지만 *입력 데이터의 품질*에서 분기하는 것이 DILAB의 핵심 차별점.

## 10. Strategic Implications — DILAB 의사결정 시사점

1. **데이터 채널 범위**: MVP는 **공개 리뷰 채널 2개(쿠팡 또는 네이버스토어 + 전문가 DB)** 로 좁히고, 채널 확장은 PMF 검증 후 단계적 확장 권장. 싱클리의 15+ 통합을 따라가는 것은 자원 분산.
2. **기술 스택 권장 출발점**:
   - 벡터 DB: MongoDB Atlas Vector Search(싱클리 검증 패턴) 또는 Pinecone/Qdrant 중 학사·교수님 인프라 가능한 옵션 검토
   - LLM: OpenAI GPT-4o-mini 또는 Anthropic Claude Haiku로 비용 절감, 도메인 fine-tuning은 PMF 후
   - 임베딩: OpenAI text-embedding-3-small 시작, 한국어 특화 모델(E5-large, BGE-M3) 후속 비교
3. **가격·GTM 가설**:
   - 학교·연구실·소규모 브랜드 대상 **freemium**(월 N건 무료) + 유료 5~15만원/월
   - 자가 가입 흐름 필수 — 싱클리의 최대 약점을 정공법으로 공략
4. **차별화 메시지**: "**양으로 답하는 싱클리, 질로 답하는 DILAB**" — 같은 RAG 골격에서 *데이터의 신뢰성*이 다르다는 단일 메시지로 정렬
5. **디자인 정책 재확인**: 정보 구조는 익숙한 패턴(좌측 네비 + 카드 KPI)을 따르되, 시각 톤은 "리서치 노트·근거 강조" 무드로 의도적 차별화 — 다른 회사가 MVP를 활용해도 시각 유사성으로 인한 분쟁 소지 최소화
6. **추가 조사 필요(Open Questions)**:
   - 한국에 "전문가 리뷰 DB"를 구축·운영하는 협력 가능 기관/협회는?
   - 도메인(예: 화장품) 외 다음 후보 도메인(가전·F&B·생활용품)의 전문가 DB 구하기 난이도는?
   - DILAB MVP를 정부 과제 제안서에 매핑할 때, 어떤 R&D 키워드(예: "도메인 특화 RAG", "휴먼 인 더 루프 LLM")로 정렬할지

3. **"같은 RAG 골격을 차용하는 것이 정말 안전한가?"** — 반대 해석: 외관·기능이 너무 유사하면 후일 분쟁 소지가 있다. *재반박*: 본 보고서는 *디자인 모방 금지*를 명시하며, 기능 매핑은 *시장 표준 UX 패턴* 수준이지 싱클리 고유 자산이 아님. 단, MVP가 외부로 노출될 때는 *시각 톤* + *데이터 큐레이션* 두 축에서 의도적 차별화가 유지되는지 별도 점검 필요.
