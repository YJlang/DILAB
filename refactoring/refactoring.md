## 0. 원칙 (Hard Rule)

- **화면에 렌더링되는 한글 문구(copy)만 교체한다.** 코드·식별자·데이터 계약은 절대 건드리지 않는다.
- 건드리지 **않는** 것: DB 컬럼/테이블명(`rating_axes`·`journey_stages`·`categories`·`chunks`), `lib/*.ts`, `app/api/**/route.ts`, 모든 주석, Modal/Supabase/Cloudflare 연동 로직, 변수·prop 이름, `wrangler.jsonc` vars.
- 교체 방향: **세부 기술명은 삭제**, 남길 부분은 *일반어인데 기술 느낌이 나는 단어*로. (교수님 지침)
- `llm_model`·`latency_ms`·`cite_type` 같은 **런타임 값 노출**도 사용자에겐 불필요 → 렌더링만 제거/순화 (데이터는 그대로 받되 화면에 안 띄움).


## 1. 용어 치환 사전 (key)

| 현재 (기술 노출) | 변경 (일반어 · 기술 느낌 유지) |
|---|---|
| `BGE-M3`, 임베딩, 벡터화, 벡터 | **AI 의미 분석**, 의미 기반 검색 |
| `BERTopic`, `UMAP`, `HDBSCAN`, 클러스터링 | **AI 주제 자동 분류**, 주제 묶음 |
| `DeepSeek`, `{llm_model}`, LLM 합성 | **생성형 AI**, AI 리포트 생성 |
| `Modal` 서버리스 함수 · 분석 큐 | **클라우드 분석 엔진**, 백그라운드 분석 |
| `Supabase` 실시간 조회 | **실시간 데이터** (또는 문구 삭제) |
| `RAG`, 출처 추적 RAG | **출처 추적 분석**, 근거 기반 답변 |
| 청크 / `chunks` / evidence chunk / 근거·출처 청크 | **근거 구절**, 근거 문장, 출처 |
| 네이버 수집 | 공개 후기 수집 (이미 "공개 후기"로 되어 있음 — 재확인) |
| `categories / rating_axes / journey_stages`, `JSONB`, `domains.metadata` | 문구 전체 삭제 (순수 내부 구조) |
| `zero-shot 분류` | AI 자동 분류 |
| `INCLab` | 삭제 |
| `latency_ms` (응답시간 ms) | 삭제 (사용자 불필요) |
| `cite_type` = `expert`/`public` (영문) | 전문가 / 일반 (한글 라벨) |

## 3. 손대지 않는 것 (data contract / 내부)

- `lib/data.ts`, `lib/rag.ts`, `lib/embeddings.ts`, `lib/supabase.ts`, `lib/cf-env.ts`, `lib/types.ts`
- `app/api/ask/route.ts`, `app/api/analyze/route.ts` (주석 포함)
- DB 컬럼/테이블/RPC 이름: `chunks`, `rating_axes`, `journey_stages`, `categories`, `match_chunks`, `domains.metadata`
- 변수·prop·함수 이름 (`totalChunks`, `chunks={items}`, `data.llm_model` 등 — *값을 화면에 안 띄울 뿐* 코드는 유지)
- `wrangler.jsonc` vars (`DEEPSEEK_*`, `MODAL_*`, `LLM_MODEL` …)

→ 즉 **UI 텍스트 노드만 교체**. 빌드·배포·연동 무손상.
